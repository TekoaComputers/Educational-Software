# Asset diff viewer — `tools/asset_diff/`

Side-by-side browser of every per-app asset, original vs ported. Useful
for spotting case-mismatch bugs, missing assets, and quality regressions
across image / audio / video.

## Files

- `build_manifest.py` — Python scanner. Walks each app's untracked
  original dir at the repo root (`Brahot/`, `Hemed/`, …) and the
  matching ported `<site>/assets/<App>/`, pairs files by
  case-insensitive basename and category (image / audio / video),
  emits a `manifest.json` (~5 MB, gitignored).
- `index.html` — static viewer. Loads `manifest.json` and renders one
  pair per row with filters (category, status, free-text). Image side
  is an `<img>`, audio is `<audio controls>`, video is `<video controls>`.

## Run

```bash
python tools/asset_diff/build_manifest.py    # regenerate after asset changes
python3 -m http.server 8080                  # browsers block file:// cross-dir loads
# open http://localhost:8080/tools/asset_diff/
```

## Status colors

- **green** — pair matched on both sides
- **amber** — original-only (asset was in the VB6 source but not ported)
- **red** — ported-only (the file exists in the ported tree but has no
  matching original — often expected for files generated by the porter)

## When to use

- After porting a new app — sanity check that every original asset
  landed somewhere in the ported tree.
- After regenerating audio via the wav→mp3 pipeline — confirm the .mp3s
  exist alongside the .wavs they were derived from.
- When a 404 shows up at runtime — search for the file in the viewer
  to spot a case-mismatch (CLAUDE.md "Asset case is unstable" gotcha).
